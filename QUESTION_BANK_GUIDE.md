## 配置文件说明

题库配置文件为 `question_bank_config.json`，采用JSON格式存储所有面试题目。

## 如何自定义题库

### 1. 启用/禁用整个大类

在 `question_bank_config.json` 中找到对应的大类，修改 `enabled` 字段：

```json
"backend": {
  "name": "后端组件",
  "enabled": false,  // 设置为false禁用整个后端组件类
  "categories": { ... }
}
```

### 2. 启用/禁用具体分类

在对应大类下找到具体分类，修改 `enabled` 字段：

```json
"backend_docker": {
  "name": "Docker",
  "enabled": false,  // 设置为false禁用Docker题目
  "count": 59,
  "questions": [ ... ]
}
```

### 3. 面试时的题目选择逻辑

系统会根据以下规则选择题目：

1. **技能匹配**：根据简历中识别的技能栈，优先选择相关分类的题目
2. **难度分级**：根据工作年限自动调整题目难度（1-3年：难度3；3-5年：难度4；5年+：难度5）
3. **随机抽取**：从启用的分类中随机选取指定数量的题目
4. **追问机制**：部分题目有50%概率触发深度追问

### 4. 添加自定义题目

可以在对应分类的 `questions` 数组中添加新题目，格式如下：

```json
{
  "id": "custom-001",
  "text": "你的自定义题目",
  "category": "database_mysql",
  "difficulty": 3,
  "type": "technical"
}
```

字段说明：
- `id`: 唯一标识符
- `text`: 题目内容
- `category`: 分类标识（必须与所属分类一致）
- `difficulty`: 难度等级（1-5，1最简单，5最难）
- `type`: 题目类型（`technical`技术题/`design`设计题/`project`项目题）

## 推荐配置

### 后端开发面试（标准）
- ✅ 编程语言：Go语言
- ✅ 数据库：MySQL、Redis
- ✅ 消息队列：Kafka
- ✅ 后端组件：Docker、Nginx
- ✅ 架构设计：微服务、分布式系统
- ✅ 计算机基础：Linux、计算机网络
- ✅ 系统设计：启用

### 校招/初级面试
- ✅ 编程语言：Go语言
- ✅ 数据库：MySQL、Redis（基础题）
- ✅ 计算机基础：Linux、计算机网络、操作系统
- ⏭️ 架构设计：暂不启用（难度较高，建议有工作经验后再学）
- ⏭️ 系统设计：暂不启用（需要实际项目经验支撑）

### 高级/架构师面试
- ✅ 所有分类全部启用
- ✅ 重点：系统设计、分布式系统、微服务

## 注意事项

1. 修改配置后需要重启服务才能生效
2. 建议保留原始配置文件备份
3. 每类题目建议至少保留10道以上，以保证随机抽取的多样性
4. 自定义题目的 `id` 不能重复
